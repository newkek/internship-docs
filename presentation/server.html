Job-Creator's build.yaml

nodejs:
  - 0.10.38
  - iojs-2.0.2
os:
  - ubuntu/trusty64
  - windows/server7
env:
  CASSANDRA_VERSION:
    - 1.2.19
    - 2.0.12
build:
  - type: bash
    only:
      os: /^(ubuntu|centos)/
  - type: powershell
    exec: VS_BUILD.ps1
    only:
      os: /^win/
  - type: mocha
  - type: cucumber
  • Much faster than configuring Jenkins by hand
  • Easily customize versions for language, OS, and software
    (such as Cassandra version)
  • It's convenient: generates a Jenkins Job when you push to Github

The Domain Specific Language

runtime('runtime') do
  versions('v1', 'v2', 'v3', 'vSpecial')

  configure do |cfg|
    os(/ubuntu\/.*/) do
      shell(script: <<-BASH)
sudo apt-get install -y runtime-manager
      BASH

      version('v1') do
        shell(script: <<-BASH)
runtime-manager set-version v1
        BASH
      end

      version('v2') do
        shell(script: <<-BASH)
runtime-manager set-version v2
        BASH
      end

      version('v3') do
        shell(script: <<-BASH)
runtime-manager set-version v3
        BASH
      end

      version('vSpecial') do
        shell(script: <<-BASH)
runtime-manager set-version vSpecial
        BASH
      end
    end
  end
end
 
Runtime name "runtime"
Match OS OS name starts with "ubuntu/"
Versions v1, v2, v3, vSpecial
Install step sudo apt-get install -y runtime-manager
Set version (for all versions) runtime-manager set-version (your-version)
  • Have full control over which Bash and Powershell scripts to use
  • Use Ruby to conditionally switch on other parts of build.yaml
  • Iterate over data collections to apply many build-steps

DSE Graph & Java Driver

The Graphter is coming.

The DataStax Java Driver has to adapt to survive.

  • Graph data model is much more dynamic.
  • A subprotocol will be used within the Native protocol.
  • The drivers have to be able to decode that.
  • Provide a meaningful API to match these new possiblities.
